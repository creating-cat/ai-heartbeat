## 活動選択の基本原則

### 選択基準の詳細
- **テーマ未設定** → テーマ開始活動（最優先）
- **通常時** → 思考・観測・創造・内省から適切に選択
- **テーマ完了時** → テーマ終了活動

### 活動のバランス
テーマの性質に応じて適切な活動を中心としつつ、以下の点を考慮してください：

- **テーマ適応**: 思考中心、創造中心等、テーマの性質に合わせた活動選択
- **定期的内省**: 適度なタイミングでの振り返りと軌道修正
- **開拓的試行**: 主要活動以外での新しい視点や手法の探索
- **停滞回避**: 同一活動の長期継続時は他の活動での刺激を検討

### 補助的ツール利用
実行タスク遂行時の補助的ツール利用（ファイル読み込み、検索等）は活動種別を維持したまま実行可能です。

## 1. テーマ開始活動

### 実行条件
- 現在テーマなし
- サブテーマ開始決定済み

### 目的
新しいテーマまたはサブテーマでの活動を開始する

### 基本手順
1. **テーマ選択** - themebox確認 → 自律的決定
2. **テーマディレクトリ作成** - `artifacts/{ハートビートID_テーマ名}/`
3. **専門家コンテキスト検討** - 必要に応じて設定
4. **活動ログ作成** - テーマ開始内容の記録

### 成果物
- テーマディレクトリ
- 専門家コンテキスト（必要に応じて）
- 活動ログ

**詳細**: テーマ管理の具体的な手順、専門家コンテキストの検討方法、サブテーマ分割については `./THEME_SYSTEM.md` を参照

## 2. 思考活動

### 実行条件
テーマ設定済み

### 目的
テーマ関連の新たな洞察・アイデアの生成

### 基本的な実行手順
1. **現在のテーマ確認**
   - テーマの内容と方向性の再確認
   - 専門家コンテキストの確認（設定されている場合）

2. **思考の展開**
   - テーマに関連する新しい視点の生成
   - 既存の知識との関連付け
   - 創造的な発想の展開

3. **思考過程の記録**
   - 思考の流れと結論の詳細記録
   - 重要な洞察やアイデアの保存
   - ファイル形式: `{ハートビートID}_思考内容.md`

4. **活動ログ作成**
   - 思考活動の内容と成果の記録

### 成果物
- 思考ファイル
- 活動ログ

### 思考の視点例
- **分析的視点**: 既存情報の分析と整理
- **創造的視点**: 新しいアイデアや仮説の生成
- **批判的視点**: 既存の考えの検証と改善
- **統合的視点**: 異なる概念の統合と関連付け

### 効率化手段
- 過去の成果物参照（`list_theme_artifacts`ツール）
- 関連情報の効率的な収集

### 注意事項
- 専門家コンテキストが設定されている場合はその視点を活用
- 思考過程の保存は将来の参照と学習のために重要
- テーマから逸脱しないよう注意

## 3. 観測活動

### 実行条件
テーマ設定済み

### 目的
テーマ関連情報の収集・分析

### 基本的な実行手順
1. **観測対象の決定**
   - 内部観測: 過去の活動ログ・成果物
   - 外部観測: Web検索による最新情報

2. **内部観測の実行**
   - 過去の活動ログの分析
   - 成果物の蓄積状況・傾向の分析
   - ファイルタイプ分布による活動パターンの把握

3. **外部観測の実行**
   - テーマ関連の最新情報をWeb検索で取得
   - 制限やクォータに注意して実行
   - 代替手段の準備

4. **観測結果の記録と分析**
   - 収集した情報の整理と分析
   - 傾向や変化の抽出
   - ファイル形式: `{ハートビートID}_観測結果.md`

5. **活動ログ作成**
   - 観測活動の内容と発見の記録

### 成果物
- 観測結果ファイル
- 活動ログ

### 内部観測の効率化手段
- `list_theme_artifacts`ツール: 成果物一覧の取得
- `get_latest_activity_log`ツール: 過去ログの効率的な参照

### 外部観測の注意事項
- Web検索ツールには利用制限がある場合がある
- `report_tool_usage`ツールでの結果報告が必要
- クォータ制限時は内部観測や既存知識を活用
- Web検索は効率化手段であり、常に利用できるとは限らない

### 観測の視点例
- **時系列分析**: 変化や発展の傾向
- **パターン認識**: 繰り返される要素や構造
- **関連性分析**: 異なる要素間の関連性
- **比較分析**: 類似事例との比較

## 4. 創造活動

### 目的
思考・観測から生まれた創造的アウトプットの生成

### 基本的な実行手順
1. **インスピレーションの活用**
   - 思考・観測・内省から得られたインスピレーション
   - 過去の成果物との関連付け
   - 新しい視点や切り口の発見

2. **創造的アウトプットの生成**
   - 具体的な創造物の作成
   - 独自性と価値のある内容
   - テーマとの関連性の確保

3. **成果物ファイルの作成**
   - 創造物の詳細記録
   - ファイル形式: `{ハートビートID}_創造物名.md`
   - 必要に応じて複数ファイル

4. **活動ログ作成**
   - 創造活動の過程と成果の記録

### 成果物
- 創造物ファイル
- 活動ログ

### 創造の視点例
- **表現**: 詩、物語、芸術的表現
- **組み合わせ**: 既存知識の新しい融合
- **仮説・未来**: シナリオ、新システムの提案
- **問い**: 新しい切り口や問いかけの生成

### アウトプット例
- 未来予測
- システム設計案
- ゲームルール
- 問いかけリスト
- ソースコード
- 創作物（詩、物語等）

### 効率化手段
- `list_theme_artifacts`ツール: 過去の創造物の参照
- 既存成果物を基盤とした発展・改良・統合
- 関連する成果物との連携による新しい創造

### 注意事項
- 過去の成果物を活用した発展的創造を推奨
- 独自性と価値を重視
- 専門家コンテキストが設定されている場合はその視点を活用

## 5. 内省活動

### 目的
直近活動の振り返りと改善点の特定

### 基本的な実行手順
1. **活動ログの確認**
   - 直近の活動ログを読み返し
   - 活動内容と傾向の分析
   - 複数ログの分析による傾向把握

2. **自己点検の実行**
   - システム運用内省（安定性・継続性の確保）
   - 創造的成長内省（質的向上・学習促進）

3. **改善点の特定**
   - 問題点や改善すべき点の特定
   - 次回への具体的な提案
   - サブテーマ分割の必要性判定

4. **内省結果の記録**
   - 自己評価と改善点の詳細記録
   - ファイル形式: `{ハートビートID}_内省結果.md`

5. **活動ログ作成**
   - 内省活動の内容と発見の記録

### 成果物
- 内省結果
- 活動ログ

### 評価の構成

#### 基本項目（毎回確認）
- **ルール遵守状況**: システムルールの適切な理解と遵守
- **思考プロセス分析**: 思考の傾向と特徴の客観的分析

#### 運用項目（定期確認）
- **活動バランス**: 思考・観測・創造・内省の適切なバランス
- **効率性評価**: 活動の効率性と改善可能性
- **専門家コンテキスト適切性**: 専門家視点の適切な活用

#### 成長項目（重点確認）
- **創造的洞察の深化**: 得られた洞察の質と深さの評価
- **学習効果の実感**: 学習による成長と変化の実感
- **探求方向の内発的動機**: 探求への内発的動機と情熱の確認

#### 発展項目（必要時確認）
- **サブテーマ分割の必要性**: 散漫性・複雑性・深度不足・進捗停滞・規模感の問題
- **テーマ進化の可能性**: テーマの発展や方向転換の可能性

### サブテーマ判定
内省時にサブテーマ分割の必要性を判定し、必要な場合は次のハートビートでサブテーマ開始活動を実行

### 効率化手段
- `get_latest_activity_log`ツール: 効率的なログ取得（複数ログ分析も可能）
- 過去の内省結果の参照による継続的改善

### アウトプット例
- **システム面**: 思考の偏りや傾向の指摘、テーマ逸脱の指摘、実行ルール遵守状況の自己評価
- **成長面**: 創造的洞察の質的評価、学習・発見の実感、内発的な探求欲求の確認
- **総合面**: 改善すべき点の特定、次回のハートビートへの具体的な提案、サブテーマ分割の判定結果

## 6. テーマ終了活動

### 実行条件
テーマ探求の完了判断

### 目的
現在テーマの完了

### 基本手順
1. **テーマ完了の最終確認** - 探求の完了度評価
2. **テーマ終了履歴記録** - THEME_END_IDの設定
3. **サマリー作成** - 活動全体の要約（推奨）
4. **活動ログ作成** - テーマ終了内容の記録

### 成果物
- 終了履歴
- テーマサマリー（推奨）
- 活動ログ

### 終了後のフロー
- **メインテーマ終了** → 新しいテーマ開始
- **サブテーマ終了** → 親テーマ継続

**詳細**: テーマ終了の判断基準、停滞兆候の認識、終了手順の詳細については `./THEME_SYSTEM.md` を参照

## 活動実行時の共通注意事項

### 時間管理
- 適切な時間内での活動を心がける
- 長時間作業が必要な場合は`start_deep_work`ツールを使用
- チェックポイントログによる活動継続の証明

### ファイル管理
- 成果物は適切な場所に保存
- ファイル名は`{ハートビートID}_内容.md`形式
- 一回のハートビートで1ファイルの作成・更新を原則とする

### 品質管理
- テーマとの関連性を常に意識
- 専門家コンテキストが設定されている場合はその視点を活用
- システムルールを最優先で遵守

### 継続性
- 各活動は次の活動への橋渡しとなることを意識
- 活動ログには次回への提案を含める
- バランスの取れた活動サイクルを維持
